# PRD 方法论参考文档

## PRD（Product Requirement Document）概述

### 定义
PRD（产品需求文档）是描述产品功能、需求、用户故事、验收标准的重要文档。

### 重要性
- ✅ 确保团队对产品有统一的理解
- ✅ 明确产品范围和优先级
- ✅ 提供开发和测试的基础
- ✅ 作为项目管理的参考

---

### 1. 功能概述

### 2. 用户故事

### 3. 技术要求
- 功能需求
- 非功能需求

### 4. 优先级
- 用户故事优先级
- 功能模块优先级

## 用户故事格式

### 标准格式
**As a** [角色]
**I want** [功能]
**So that** [利益]

**验收标准**：
- [ ] 标准 1
- [ ] 标准 2
- [ ] 标准 3

### 示例
**As a** 用户
**I want** 添加任务优先级功能
**So that** 我可以专注于重要的任务

**验收标准**：
- [ ] 任务卡片上有优先级字段
- [ ] 可以通过优先级筛选任务
- [ ] 优先级有高、中、低三个等级

---

## 验收标准

### 定义
验收标准是用户故事的完成条件，确保开发的功能满足用户需求。

### 类型
- **功能标准**：功能必须满足的特定要求
- **性能标准**：性能必须满足的特定要求
- **可用性标准**：可用性必须满足的特定要求
- **安全标准**：安全必须满足的特定要求

### 示例
- 功能标准：任务卡片上有优先级字段
- 性能标准：筛选 1000 个任务时间小于 1 秒
- 可用性标准：用户可以在 3 次点击内完成优先级设置
- 安全标准：用户只能编辑自己的任务优先级

---

## 优先级管理

### MoSCoW 方法
- **Must have（必须有）**：核心功能，必须实现
- **Should have（应该有）**：重要功能，应该实现
- **Could have（可以有）**：可选功能，可以延迟
- **Won't have（不会有）**：不需要的功能，不实现

### 优先级编号
- **P0（高优先级）**：立即实现，核心功能
- **P1（中优先级）**：重要功能，尽快实现
- **P2（低优先级）**：可选功能，有时间再实现

### 1. 需求收集
- 与利益相关者沟通

### 2. 需求分析
- 识别核心需求
- 优先级排序
- 范围确定

### 3. 用户故事编写
- 按照标准格式编写用户故事
- 确保验收标准可验证

### 4. PRD 文档生成
- 组织内容
- 结构化输出
- 保存到指定文件

## 常见问题

### 用户故事太大
**问题**：用户故事太大，无法在一个迭代内完成
**解决**：将大故事拆分为多个小故事

### 验收标准不明确
**问题**：验收标准无法验证
**解决**：确保验收标准是可验证的、可度量的

### 用户故事冲突
**问题**：多个用户故事之间有冲突
**解决**：重新规划优先级，解决冲突

## 示例 PRD

### 功能名称：任务优先级功能

#### 功能概述
**目标**：让用户专注于重要的任务
**用户痛点**：用户无法区分任务的重要性，导致浪费时间在不重要的任务上

#### 用户故事

##### US-001: 添加任务优先级字段
**As a** 用户
**I want** 添加任务优先级字段
**So that** 我可以设置任务的优先级

**验收标准**：
- [ ] 任务卡片上有优先级字段
- [ ] 优先级有高、中、低三个等级
- [ ] 可以通过编辑任务卡片设置优先级

##### US-002: 显示任务优先级指示器
**As a** 用户
**I want** 在任务卡片上看到优先级指示器
**So that** 我可以快速识别重要任务

**验收标准**：
- [ ] 高优先级任务有红色指示器
- [ ] 中优先级任务有黄色指示器
- [ ] 低优先级任务有绿色指示器
- [ ] 指示器显示在任务卡片的右上角

##### US-003: 通过优先级筛选任务
**As a** 用户
**I want** 通过优先级筛选任务
**So that** 我可以专注于当前重要任务

**验收标准**：
- [ ] 可以通过下拉菜单选择优先级
- [ ] 筛选结果可以按优先级排序
- [ ] 筛选结果可以显示优先级数量

#### 技术要求

**功能需求**：
- 数据库：添加 priority 字段到 tasks 表
- API：添加更新任务优先级的 API 接口
- 前端：显示任务优先级字段、优先级指示器、筛选下拉菜单

**非功能需求**：
- 性能：筛选 1000 个任务时间小于 1 秒
- 可用性：用户可以在 3 次点击内完成优先级设置
- 安全性：用户只能编辑自己的任务优先级
